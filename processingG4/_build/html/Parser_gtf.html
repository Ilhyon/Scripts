Parser_gtf¶

Copyright:
Copyright Universite of Sherbrooke, departement of biochemistry and departement of computation.
Date:
May 2019
Description:
This script is principaly a library containing many methods to import a gtf file. It can be imported as a dictionary or as a dataFrame. While imported as dictionary, intron are computed and added to the dictionnary. An ouput containing introns information can be created.
Data availability:
  • gtf file via Ensembl ftp.

Parser_gtf.computesCoordLastRank(strand, start, end)¶

Computes the coordinates of the last intron of a transcript.

Depending of the strand, the end of the last intron is computed. It corresponds to the nucleotids before the last exon start.

Parameters:
  • strand (string) – strand of a transcript.
  • start (integer) – chromosomal start of the exon.
  • end (integer) – chromosomal end of the exon.
Returns:

intronEnd, end of the last exon
Return type:

integer
Parser_gtf.computesCoordOther(strand, start, end)¶

Computes the coordinates of midle intron of a transcript.

With the coordinates of a “midle” exon we can compute the end of the previous exon and then the start of the next intron.

Parameters:
  • strand (string) – strand of a transcript.
  • start (integer) – chromosomal start of the exon.
  • end (integer) – chromosomal end of the exon.
Returns:

intronStart and intronEnd, end of the previous intron and start of the next intron.
Return type:

integer
Parser_gtf.computesCoordRank1(strand, start, end)¶

Computes the coordinates of the first intron of a transcript.

Depending of the strand, the start of the first intron is computed. It corresponds to the nucleotids next to the first exon end.

Parameters:
  • strand (string) – strand of a transcript.
  • start (integer) – chromosomal start of the exon.
  • end (integer) – chromosomal end of the exon.
Returns:

intronStart, start of the first exon
Return type:

integer
Parser_gtf.countIntron(dicoTr, sp)¶

Counts the length of intron and write them in a file.

Some transcript have intron of 1 nucleotides. We computes the length of introns to see for each species what’s the range of the intron length.

Parameters:
  • dicoTr (dictionnary) – contains all transcript of a specie and all features in it (exon, utr, intron and coords).
  • sp (string) – name of the specie.
Parser_gtf.getIntron(dicoTr)¶

Gets all intron of a specie transcriptom.

We browse all transcripts of a specie and computes all intron coordinates. Coordinates will be stored as chromosomal coordinates. This mean that the start will always be inferior compare to the end.

Parameters:dicoTr (dictionary) – contains all transcripts from a psecie and all features linked to it : UTR, exon, intron.
Returns:dicoTr, updated with introns.
Return type:dictionary

Parser_gtf.importGTFdf(filename)¶

Imports a gtf file into a dataframe.

Read the gtf file into a csv, then change the column names. NI goes for ‘Not Important’, it’s column I will not use so I delete them. I also retrieve only what I need from attributes and then dilate it.

Parameters:filename (string) – name of the gtf file.
Returns:df, contains all transcripts feature for a species.
Return type:dataFrame
Parser_gtf.parseDF(df)¶

Parses a dataframe to retrieve only usefull data for me.

We only kept soem features : exon, CDS, 5UTR and 3UTR. From attributes we only kept : Transcript id, Biotype of the transcript and the type of the transcript (coding or not).

Parameters:df (dataFrame) – contains all transcripts feature for a species but non parsed.
Returns:dfTmp, parsed df.
Return type:dataFrame

Parser_gtf.setCoordReverseStrand(dico)¶

Change the coords depending on the strand.

I choosed to get chromosomal coords on my file. It will always be the smaller coords before the superior one. So for a reverse feature the start and the end are inversed.

Parameters:dicoTr (dictionary) – contains all transcript and its feature.
Returns:dicoTr, updated with good coordinates.
Return type:dictionary
